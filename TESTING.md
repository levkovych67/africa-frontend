## Діаграми потоків

### Покупець — шлях від каталогу до замовлення

```mermaid
flowchart TD
    A([Головна сторінка]) --> B[Каталог товарів]
    B --> C[Фільтр / пошук]
    B --> D[Клік на товар]
    D --> E[Сторінка товару]
    E --> F{Вибір варіанту}
    F -->|Є в наявності| G[Додати в кошик]
    F -->|Розпродано| H[Кнопка заблокована]
    G --> I[Кошик відкривається]
    I --> J[Оформлення замовлення]
    J --> K[Крок 1: Контакти]
    K --> L[Крок 2: Доставка]
    L --> M[Крок 3: Оплата]
    M --> N{Метод оплати}
    N -->|Накладний платіж| O[Оформити]
    N -->|Онлайн Monobank| P[Перейти до оплати]
    O --> Q([Сторінка успіху])
    P --> R[Monobank]
    R --> S([Трекінг замовлення])
    Q --> S
```

### Адмін — управління магазином

```mermaid
flowchart TD
    A([Логін]) --> B[Панель статистики]
    B --> C{Бічна панель}
    C --> D[Товари]
    C --> E[Замовлення]
    C --> F[Артисти]
    C --> G[Адміністратори]

    D --> D1[Список] --> D2[Створити]
    D1 --> D3[Редагувати]
    D1 --> D4[Видалити]

    E --> E1[Список] --> E2[Деталі]
    E2 --> E3[Змінити статус]
    E3 -->|Відправлено| E4[Ввести ТТН]

    F --> F1[Список] --> F2[Створити]
    F1 --> F3[Редагувати]
    F1 --> F4[Видалити]

    G --> G1[Список] --> G2[Створити]
    G1 --> G3[Видалити]
```

### Статуси замовлення

```mermaid
stateDiagram-v2
    [*] --> ОЧІКУЄ_ОПЛАТИ : Онлайн оплата
    [*] --> НОВЕ : Накладний платіж
    ОЧІКУЄ_ОПЛАТИ --> НОВЕ : Оплата отримана
    ОЧІКУЄ_ОПЛАТИ --> СКАСОВАНО : Таймаут
    НОВЕ --> ПІДТВЕРДЖЕНО : Адмін підтвердив
    ПІДТВЕРДЖЕНО --> ВІДПРАВЛЕНО : + ТТН
    ВІДПРАВЛЕНО --> ДОСТАВЛЕНО : Отримано
    НОВЕ --> СКАСОВАНО : Скасовано
    ПІДТВЕРДЖЕНО --> СКАСОВАНО : Скасовано
```
